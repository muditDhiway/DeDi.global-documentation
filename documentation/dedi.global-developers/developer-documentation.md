# Developer documentation

This contains the documentation for the Dedi global. The docs are organized into topic files (Overview, Standard APIs, Guides, Quickstart, Resources). Each folder contains one or more Markdown files that explain concepts, show examples, and provide step-by-step instructions.

### Table of contents

* Overview (`1. overview`)
* Standard APIs (`2. standard-apis`)
* Guides (`3. guides`)
* Quickstart (`4. quickstart`)
* Resources (`5. resources`)

### Overview

The Overview section introduces Dedi and describes the architecture. See the files in `1. overview` for:

* `1. overview/1. intro.md` — high-level introduction
* `1. overview/2. architecture.md` — architecture and design notes

### Standard APIs

This section documents the standard APIs defined by the Dedi protocol. It explains lookup, query, versioning, and other core API behaviors. Files live in `2. standard-apis`.

### Guides

Practical how-tos and walkthroughs are in the Guides folder (`3. guides`). Topics include domain verification, schema design, bulk upload, delegation, and state management.

### Quickstart

The Quickstart - `4. quickstart` explains the available environments and the exact flow to set up a local registry and try Dedi.

### Resources

Additional reference material (FAQ, glossary, roadmap) lives in `5. resources`.

### Contributing

If you find typos or want to improve the docs, please open a pull request or file an issue with suggested changes.
